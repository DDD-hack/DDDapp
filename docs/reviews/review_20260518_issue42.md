## 🟠 Major Issues

### [M-1] `recordCommit` のタイムアウトが `fetchHeartRate` と共用されている

**ファイル**: `hooks/main.go:156`
**問題**: `recordCommit` が `timeout`（3秒）を使っている。コミット記録の失敗はユーザー体験に影響しないのに、3秒のブロックが発生しうる。recordCommit はベストエフォートで良いので、タイムアウトを短く（1秒以下）するか、goroutine で非同期にすべき。

```go
// ❌ 現在: コミット判定と同じ3秒タイムアウト
client := &http.Client{Timeout: timeout}
```

```go
// ✅ 修正案: 短いタイムアウト
client := &http.Client{Timeout: 1 * time.Second}
```

または goroutine で完全に非同期化：
```go
go recordCommit(bpm, "accepted")
os.Exit(0) // goroutine が終わる前に Exit するので実質fire-and-forget
```

ただし `os.Exit` の前に goroutine は走り切らないため、素直に短いタイムアウトにする方が確実。

### [mn-1] `commit_hash` が常に空文字列

**ファイル**: `hooks/main.go:151`
**問題**: pre-commit hook はコミット前に実行されるためハッシュがまだ存在しないが、記録として空文字列が入り続ける。`NULL` の方がスキーマ設計と一致している（スキーマは `commit_hash TEXT`、NOT NULL なし）。

```go

```

```go
// ✅ 修正案: nil を送ると JSON の null になり DB に NULL が入る
"commit_hash": nil,
```

ハッカソンスコープでは実害なし。認識として記録。
