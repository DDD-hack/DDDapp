## 🟡 Minor Issues

### [mn-1] `scanner.Err()` を確認していない

**ファイル**: `hooks/main.go:202`
**問題**: `bufio.Scanner` は I/O エラーが発生した場合も `Scan()` が `false` を返す。`scanner.Err()` を確認しないと、ファイル読み取り途中のエラーを EOF と区別できない。実害は小さい（フォールバックで動き続ける）が、Go の慣例に反する。

```go
// 現在のコード
for scanner.Scan() { ... }
return 0, false
```

```go
// ✅ 修正案
for scanner.Scan() { ... }
if err := scanner.Err(); err != nil {
    // I/O エラーでも fail-open でフォールバック
    _ = err
}
return 0, false
```

エラーを無視する場合は `_ = err` で明示することで、意図的な握りつぶしであることを示せる。

### [mn-2] 無効な `threshold_bpm` 値がサイレントスキップされる

**ファイル**: `hooks/main.go:198`
**問題**: `threshold_bpm = abc` のように不正な値が書かれた場合、警告なしにスキップされてデフォルト120が使われる。環境変数 `DDD_THRESHOLD_BPM` は不正値に警告を出すのに、設定ファイルは出さない（挙動が非対称）。

```go
// ✅ 修正案
val := strings.TrimSpace(parts[1])
if n, err := strconv.Atoi(val); err == nil && n > 0 {
    return n, true
} else if err != nil {
    // 呼び出し元で warn できないため stderr に出す
    fmt.Fprintf(os.Stderr, "⚠ ~/.ddddrc: invalid threshold_bpm=%q, ignored\n", val)
}
```
