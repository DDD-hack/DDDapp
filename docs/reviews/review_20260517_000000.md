## 🟠 Major Issues

### [M-1] Average() でスライスのメモリが解放されない（メモリ成長）

**ファイル**: `daemon/internal/hrm/hrm.go:34`
**問題**: `recent := b.samples[:0]` は元のバッキング配列を再利用するため、古いサンプルが GC されない。大量サンプルが来た後も容量を保持し続ける。

```go
// ❌ 現在のコード（元の大きいスライスを使い回す）
recent := b.samples[:0]
for _, s := range b.samples {
    if s.recordedAt.After(cutoff) {
        recent = append(recent, s)
    }
}
b.samples = recent
```

```go
// ✅ 修正案（新しいスライスに移して古い容量を解放）
var recent []sample
for _, s := range b.samples {
    if s.recordedAt.After(cutoff) {
        recent = append(recent, s)
    }
}
b.samples = recent
```

**理由**: Apple Watch は 1Hz で送信する。10 秒ウィンドウなので最大 10 サンプルに収束するが、バースト送信や再接続時に一時的にスライスが膨らんだ後も容量を保持し続ける。ハッカソンの短時間デモなら実害は少ないが、規約上のメモリリークに相当する。

---

### [M-2] WebSocket の CheckOrigin が全許可（開発用のまま本番に出やすい）

**ファイル**: `daemon/internal/api/handler.go:12`
**問題**: `CheckOrigin: func(r *http.Request) bool { return true }` はすべてのオリジンを許可する。localhost のみで動くデーモンなので実害は薄いが、CLAUDE.md のセキュリティチェック観点として記録する。

```go
// ❌ 現在のコード
var upgrader = websocket.Upgrader{
    CheckOrigin: func(r *http.Request) bool { return true },
}
```

```go
// ✅ localhost のみ許可する場合（ハッカソンでは任意）
var upgrader = websocket.Upgrader{
    CheckOrigin: func(r *http.Request) bool {
        origin := r.Header.Get("Origin")
        return origin == "" || strings.HasPrefix(origin, "http://localhost")
    },
}
```

**理由**: iPhone アプリ（同一 LAN）からの接続なので origin チェックは難しい。ハッカソンスコープでは現状のままでよいが、認識しておくべき点として Major に挙げる。
