## 🟡 Minor Issues

### [mn-1] `Buffer.Add()` が bpm ≤ 0 を受け入れる

**ファイル**: `daemon/internal/hrm/hrm.go:37`
**問題**: 不正な BPM のガードはハンドラ側（`payload.BPM > 0`）にしかない。Buffer 自体は 0 以下の値を拒否しないため、将来 Buffer を直接使うコードやテストで誤った平均が出る可能性がある。

```go
// ❌ 現在のコード（ガードなし）
func (b *Buffer) Add(bpm int) {
    b.mu.Lock()
    defer b.mu.Unlock()
    now := b.now()
    b.pruneLocked(now.Add(-windowDuration))
    b.samples = append(b.samples, sample{bpm: bpm, recordedAt: now})
}
```

```go
// ✅ 修正案
func (b *Buffer) Add(bpm int) {
    if bpm <= 0 {
        return
    }
    b.mu.Lock()
    defer b.mu.Unlock()
    now := b.now()
    b.pruneLocked(now.Add(-windowDuration))
    b.samples = append(b.samples, sample{bpm: bpm, recordedAt: now})
}
```

### [mn-2] 並行テストで Add と Average が真に同時に動かない可能性

**ファイル**: `daemon/internal/hrm/hrm_test.go:100`
**問題**: 現在のテストは「Add の goroutine を50個起動 → Average の goroutine を50個起動」という順番。Add が全部終わってから Average が始まる可能性があり、真の並行アクセスが起きない場合がある。

```go
// ❌ 現在のコード（2段階に分かれている）
for i := 0; i < 50; i++ { // Add を先に起動
    wg.Add(1)
    go func(bpm int) { ... buf.Add(bpm) }(i + 1)
}
for i := 0; i < 50; i++ { // Average を後で起動
    wg.Add(1)
    go func() { ... buf.Average() }()
}
```

```go
// ✅ 修正案（Add と Average を交互に起動）
for i := 0; i < 50; i++ {
    wg.Add(2)
    go func(bpm int) { defer wg.Done(); buf.Add(bpm) }(i + 1)
    go func() { defer wg.Done(); buf.Average() }()
}
```
