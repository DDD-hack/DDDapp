## 🟡 Minor Issues

### [mn-1] `time.Sleep` による非決定的な WebSocket テスト

**ファイル**: `handler_test.go:169`, `handler_test.go:200`

```go
// 現在のコード
time.Sleep(50 * time.Millisecond)
```

50ms はローカルでは十分だが、負荷の高い CI 環境では足りない場合がある。`buf.Average()` をポーリングする方が確実。

```go
// ✅ より堅牢な案
deadline := time.Now().Add(500 * time.Millisecond)
for time.Now().Before(deadline) {
    if _, ok := buf.Average(); ok {
        break
    }
    time.Sleep(5 * time.Millisecond)
}
```

現状でも実用上の問題はほぼなし。CI 設定（#19）時に flaky になるようなら対処で十分。

### [mn-2] `postCommit` ヘルパーで `json.Marshal` のエラーを無視している

**ファイル**: `handler_test.go:80`

```go
// 現在のコード
b, _ := json.Marshal(body)
```

テストヘルパーなので実害は極めて小さいが、`map[string]any` は必ず marshal できるため `_ =` が正当化される。意図を明示するなら `t.Fatal` に変更するか、あるいはそのままでも許容範囲。

```go
// ✅ 明示的な案
b, err := json.Marshal(body)
if err != nil {
    t.Fatalf("marshal: %v", err)
}
```
